# Security Policy

## Reporting a Vulnerability

Please **do not open public GitHub issues** for security vulnerabilities.

Instead, use one of the following private channels:

1. **GitHub Security Advisories** (preferred):
   - Go to the repository page → **Security** tab → **Report a vulnerability** (if enabled).
2. **Email**: send details to **contact@aditya-verma.me**.

Include as much of the following as possible:

- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept if available)
- Affected page/section and file(s) (e.g. `index.html`, `main.js`)
- Any relevant screenshots, logs, or browser/OS details

## Supported Versions

This is a static site. The **latest version on the `main` branch** (and the live deployment) is the supported version.

## Response Targets

Best effort:

- **Acknowledgement**: within 72 hours
- **Initial assessment**: within 7 days
- **Fix timeline**: depends on severity and complexity

## Notes

- Third‑party libraries are primarily loaded via CDNs. If your report concerns an upstream dependency, please also consider reporting it to the upstream maintainers.
